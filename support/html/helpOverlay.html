Warning: The useage of USIs relies on requests against the PROXI api of the major proteomics resources (jPOST, MassIVE, PeptideAtlas, PRIDE, ProteomeCentral). These APIs have at the time of writing (19.04.2021) different response formats and we limit our visualization to USIs that can be served by ProteomeCentral and PeptideAtlas. Those resources provide all information to make the USE workable.

The main focus of this tool is to compare two peptide spectra either by manual input and/or by reference via the Universal Spectrum Identifier. Alternatively, peptide spectra can be predicted via Prosit or retrieved as synthetic peptide spectra of the ProteomeTools project. This projects builds on the Interactive Peptide Spectral Annotator (IPSA) and is recommended when single spectral visualization is needed.
If a user is interested in the comparison of metabolomic spectra, please use the Metabolomics Spectrum Resolver.

Spectrum merging and scoring

Merging

Spectral similarity between two spectra (top mirror plot) can be used to address three questions: 1) Whether two spectra are identical (bottom middle), using all peaks for similarity calculation, 2) whether the top spectrum is contained in the bottom spectrum (bottom left), discarding unmatched peaks in the bottom spectrum, or 3) whether the bottom spectrum is contained in the top spectrum (bottom right), discarding unmatched peaks in the top spectrum. The spectral angle (SA) is highlighted in red for each of the three cases. Peaks used to calculate the spectra angle are highlighted in red for each case.

Limitations

  • The web tool can not be used completly independently of a service called use_proxy. This is due to the fact that not all endpoints of the USI providers (such as JPOST, MassIVE, PeptideAtlas, PRIDE and ProteomeCentral) are served via https. ProteomicsDB hosts the UniversalSpectrumExplorer and uses TLS (https) to encrypt the communication with the user and host the web application. Unfortunately, all major browsers disallow communication from originally encrypted sources to other websites that do not provide a TLS communication channel. ProteomicsDB does not want to compromise with its communication and hosting policy and provides a simple proxy that wraps requests to all endpoints. We are happy to remove this workaround when all resources embrace encrypted communication.
  • We are using the ProXI service to automatically route USI request to the correct endpoint, but as this service is still under active development, we can't guarantee 100% compatibility.
